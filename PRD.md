# PRD.md — 감성 분석 서비스 제품 요구사항 문서

## 1. 제품명
**FROM TEXT TO FEELING**

## 2. 제품 한 줄 설명
사용자가 입력한 문장을 OpenAI API로 분석해 긍정, 부정, 중립 감성을 직관적으로 보여주는 웹 서비스.

## 3. 문제 정의
초보 사용자는 긴 글이나 고객 의견의 분위기를 빠르게 파악하기 어렵다. 이 서비스는 텍스트의 감성을 자동으로 분석해 결과를 간단한 카드 형태로 보여준다.

## 4. 핵심 사용자
- 고객 리뷰를 빠르게 확인하려는 실무자
- 댓글, 설문 응답, 짧은 문장의 분위기를 파악하려는 사용자
- AI API 기반 웹 서비스를 학습하려는 비전공자 학생

## 5. 핵심 기능
| 기능 | 설명 | 필수 여부 |
|---|---|---|
| 텍스트 입력 | 사용자가 분석할 문장을 입력 | 필수 |
| 감성 분석 요청 | 백엔드가 OpenAI API에 분석 요청 | 필수 |
| 결과 표시 | 긍정 / 부정 / 중립 중 하나 표시 | 필수 |
| 신뢰도 표시 | 0~100% 백분율 표시 | 필수 |
| 분석 이유 표시 | 왜 그렇게 판단했는지 짧게 설명 | 필수 |
| 오류 메시지 | 빈 입력, API 오류, 서버 오류 표시 | 필수 |
| 분석 기록 저장 | Supabase에 입력/결과 저장 | 선택이지만 구현 권장 |

## 6. 사용자 흐름
```txt
메인 화면 접속
↓
텍스트 입력
↓
분석하기 버튼 클릭
↓
로딩 상태 표시
↓
OpenAI API 분석
↓
긍정 / 부정 / 중립 결과 표시
↓
신뢰도와 분석 이유 표시
↓
필요 시 Supabase에 기록 저장
```

## 7. 화면 요구사항
첨부 이미지의 분위기를 참고한다.

- 밝은 회색 배경
- 상단 내비게이션
- 굵고 큰 타이틀
- 중앙 정렬된 메인 영역
- 어두운 카드 또는 패널 영역
- 둥근 모서리 버튼
- 검정, 회색, 연노랑 포인트 컬러
- 미니멀하고 세련된 레이아웃

## 8. 제약 조건
- 프론트엔드는 HTML, CSS, JavaScript만 사용한다.
- 백엔드는 Node.js와 Express를 사용한다.
- OpenAI API Key는 서버에서만 사용한다.
- 프론트엔드에서 OpenAI API를 직접 호출하지 않는다.
- Supabase 연결 실패가 전체 서비스 실패로 이어지면 안 된다.

## 9. 성공 기준
- 사용자는 1분 이내에 텍스트를 입력하고 결과를 확인할 수 있다.
- 결과는 반드시 `positive`, `negative`, `neutral` 중 하나로 서버에서 정규화한다.
- 화면에는 한국어로 `긍정`, `부정`, `중립`을 표시한다.
- 오류가 발생해도 사용자가 원인을 이해할 수 있어야 한다.
