# 02_ui_design_guide.md — UI 디자인 지침

## 1. 참고 이미지 해석
첨부 이미지는 Awwwards 스타일의 미니멀 웹사이트 화면이다. 핵심 특징은 다음과 같다.

- 전체 배경은 밝은 회색
- 상단에 로고, 검색 영역, 버튼이 있는 내비게이션
- 중앙에 매우 큰 타이포그래피
- 검정과 회색 중심의 절제된 색상
- 하단에 어두운 카드형 콘텐츠 영역
- 둥근 모서리와 넉넉한 여백
- 버튼은 단순하지만 눈에 띄게 표현

## 2. 서비스 화면 방향
서비스 제목은 큰 글자로 표현한다.

예시:
```txt
FROM TEXT TO FEELING
```

부제는 짧고 쉽게 작성한다.

예시:
```txt
문장을 입력하면 AI가 감성을 분석합니다.
```

## 3. 색상 가이드
```css
--bg: #e8e8e8;
--text: #202020;
--muted: #777777;
--panel: #1f1f1f;
--panel-soft: #2b2b2b;
--line: #3a3a3a;
--point: #fff176;
--white: #ffffff;
--error: #ff6b6b;
```

## 4. 레이아웃 지침
- 화면 최대 너비는 1200px 안팎으로 제한한다.
- 상단 영역은 좌우 정렬한다.
- 메인 타이틀은 데스크톱에서 매우 크게 표시한다.
- 입력 영역과 결과 영역은 어두운 카드 안에 배치한다.
- 모바일에서는 타이틀 크기를 줄이고 카드가 세로로 쌓이게 한다.

## 5. 컴포넌트 구성
| 컴포넌트 | 설명 |
|---|---|
| Header | 로고, 설명, 상태 표시 |
| Hero | 큰 제목과 서비스 소개 |
| Input Panel | 텍스트 입력창과 분석 버튼 |
| Result Card | 감성, 신뢰도, 이유 표시 |
| Error Box | 오류 메시지 표시 |

## 6. 버튼 스타일
- 기본 버튼은 검정 배경, 흰색 글자
- 주요 CTA 버튼은 연노랑 배경, 검정 글자
- hover 시 살짝 위로 올라가는 효과 적용

## 7. 완료 기준
- 첨부 이미지처럼 넓은 여백과 큰 타이틀이 느껴져야 한다.
- 입력창, 버튼, 결과 카드가 한 화면에서 자연스럽게 보여야 한다.
- 모바일에서도 가로 스크롤이 없어야 한다.
